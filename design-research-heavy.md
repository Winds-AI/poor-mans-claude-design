Use this file only for serious design tasks, broad exploration, or when the user asks for research-heavy work.

Skip this flow for narrow fixes, text tweaks, spacing fixes, bug fixes, or small refinements.

## Goal

- Build visual judgment from real interfaces before designing.
- Use visual inspection when possible. UI is judged by how it looks.

## Tools

- Prefer `chrome-devtools` for live visual inspection.
- Use web search to find relevant references, then inspect them in the browser when possible.
- Use screenshots when they help comparison or later reuse.
- Use `ffmpeg` when motion matters and screenshots are not enough. Record short clips to inspect animation timing, state transitions, and interaction feel.

## Sources

Start with the most relevant sources for the task:

- direct competitors
- same-domain products
- adjacent-domain products with similar user behavior
- Mobbin
- Dribbble
- Pinterest
- Figma Community
- Awwwards or SiteInspire for marketing or expressive web work

Prefer real shipped products over concept shots.

## Research Flow

1. Identify product type, audience, platform, and goal.
2. Search for 3-7 strong references, not dozens.
3. Open the best references in `chrome-devtools` when possible.
4. Inspect the first screen, hierarchy, layout, density, navigation, CTA placement, trust signals, and interaction patterns.
5. If motion or interaction feel matters, record short clips with `ffmpeg` and inspect animation timing, easing, sequencing, and state transitions.
6. Save reusable screenshots or clips to `references/ideas/` when useful.
7. Extract patterns to reuse or avoid. Do not copy layouts directly.
8. Write a short synthesis before designing.

## What To Look For

- what users notice first
- page scanning path
- information density
- navigation model
- CTA prominence
- trust and credibility signals
- content grouping
- whitespace and rhythm
- tone and brand expression
- state treatment
- animation timing and restraint
- transition clarity
- interaction feedback

## Output

Summarize only what affects the design:

- patterns to keep
- patterns to avoid
- why they fit this audience and goal
- how they should influence the variants
